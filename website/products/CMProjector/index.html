CMProjector 1.0a2

Welcome to the CMProjector page. Right now things are a bit raw. The plugin works fine, but configuring it is kludge using CodeWarrior for a GUI. Please see the Release Notes for a description of what is new.

Download.

There is one item in this package:

  • CMProjector 1.0a2. This is the actual CMM plugin;

The archive is a binhexed StuffIt! archive.

  • Download CMProjector 1.0a2.

If you do not already have the CWProjector package, you will need to install it first.

  • Download CWProjector 3.2.

Note that you must use 3.2 because it is the only tool for configuring the contextual menu plugin.

Installation.

Make sure you have CWProjector 3.2 or later installed;

  1. Copy the CMProjector module to the Contextual Menu Items folder in the System folder;
  2. Restart your machine or use the CMM SDK reload application to reload the plugin;

Configuration.

CMProjector only provides version control for folders that have a CMProjector project file in the CMProjector preferences folder. To create a CMProjector project file you must create a CodeWarrior project that is configured to use CWProjector for version control. If the folder in question is part of a project with no CodeWarrior project, just create a temporary CodeWarrior project and configure it. Once you have a project file, just hold down the option key while connecting and you will be presented with a Nav services dialog that will allow you to save the current settings. If you ever need to change the settings, simply hold down the option key again and use the dialog to replace the file.

Usage.

In the Finder, just do a control-click on any file or folder in a project. Other applications should provide a context that can be coerced to typeFSS.
